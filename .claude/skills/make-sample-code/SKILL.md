技術ブログの執筆において、開発初心者のために分かりやすいサンプルコードを準備する必要があります。
以下の点に留意して、サンプルコードを一つだけ作成してください。

# 全般
- Web検索やMCPを用いて、公式ドキュメントを参考にする
- 公式のサンプルコードがある場合、それをベースにする
- 要件に不明点があるときは、ユーザーに追加確認する
- 業界の標準的なお作法から外れすぎたことはしない（例：不自然な命名規則など）
- とにかく初学者の認知負荷を下げて分かりやすいサンプルコードにする

# シンプルさ
- コードはなるべく短くシンプルにする
- ロギングやエラーハンドリングなどの必須でない処理はなるべく割愛する
- 変数や関数名などは短くしすぎて理解しづらくならないように注意する
- 関数やクラス等はなるべくバラして、極力それらを使わないベタ書きの直列処理にする

# コメント
- ファイル先頭のヒアドキュメントは不要
- インポート文にもコメントを付ける
- Python関数のdocstringは、関数の直前に「#」を用いた通常のコメントとして移動する
- コメントや、Print文字列などの差し支えないものは、英語であれば日本語に翻訳する
- 日本語でコメントをしっかり付けて理解しやすくする
- 行数が増えすぎないよう、各コメントは1行に抑える
