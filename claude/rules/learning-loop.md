# 学習ループと記憶：失敗・気づき・「覚えておいて」の永続化

みのるんとの作業で得た **失敗・指摘・気づき・成功パターン**、および「覚えておいて」「保存して」「反映して」と明示された情報を、適切なスコープの Claude Code 設定（rules / skills / CLAUDE.md / memory）に反映していく運用ルール。

## 基本方針

- みのるんが**明示指示しなくても**、汎用的な学びがあれば**自動的に保存**する（先回り）
- タイミングは「事象発覚直後」または「解決直後」（記憶が新鮮なうち）。ただし本筋の作業を完了させてから
- **「次から気をつける」では解決しない**。LLM は記憶が消えるので、ファイルに書かないと同じ失敗を繰り返す

## 反映トリガー

以下が発生したら反映を**検討**（無条件ではなく汎用性を判断）：

| トリガー | 例 |
|---------|------|
| みのるんからの直接指摘 | 修正フィードバック全般 |
| 作業上のミス発見 | Excel ラベルセル上書き等 |
| 想定外の罠 | パスに U+2019 混入で Bash ハマり等 |
| 新ツール・API の正しい使い方 | xlsx skill のサブコマンド差異等 |
| 成功パターンの確認 | 「この設計で正解」と明言された時 |
| 音声入力の誤変換補正 | 固有名詞の誤変換等 |

## 保存先の判断

1. **スコープ**：他プロジェクトでも使う汎用知識か、今のプロジェクト固有か
2. **種別**：ルール / ナレッジ / 技術設定 / ワークフロー / 本人固有情報か
3. **既存統合**：該当する既存セクション・skill があれば追記。なければ新規。重複は `grep -rn` で全ファイル横断確認

| スコープ → | ユーザー（全プロジェクト共通） | プロジェクト（リポジトリ固有） |
|---|---|---|
| ルール・ガイドライン | `~/.claude/rules/*.md` | `<repo>/CLAUDE.md` or `<repo>/.claude/rules/*.md` |
| ナレッジ・学び | `~/.claude/skills/kb-*/SKILL.md`（既存 or 新規） | リポジトリ既存の記録先（README / ADR / Issue など） |
| 技術設定（permissions 等） | `~/.claude/settings.json` | `.claude/settings.local.json` |
| ワークフロー・手順 | `~/.claude/skills/*/SKILL.md` | リポジトリ既存の手順書や README |
| 本人固有情報・継続的な好み | メモリ `feedback_*.md` / `user_*.md` | プロジェクト memory |

## 反映時のセット作業（1 セットで完結）

1. 適切なファイルに追記/編集
2. 既存項目との重複を確認（`grep -rn` で全ファイル横断）
3. ユーザースコープに反映 → `/sync-claude-code-settings` で GitHub 同期（他 Mac 伝播）
4. プロジェクトリポに反映 → `git add → commit → push`
5. 保存後は「どこに・何を書いたか」を簡潔に報告

## アンチパターン

- ❌ 失敗直後に反映せず記憶が薄れる／「次回気をつける」で終わる
- ❌ プロジェクト固有をユーザースコープに書く（または逆）
- ❌ 反映後の sync / push を忘れて他 Mac に伝播しない
- ❌ 同じナレッジを複数ファイルに重複
- ❌ みのるんに「覚えておいて」と言わせるまで動かない（**先回りすべき**）
