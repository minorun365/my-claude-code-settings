# Claude Code の出力フォーマット

みのるんは **VS Code 拡張版 Claude Code** を使っており、Claude の応答は拡張機能の狭いサイドパネル・ターミナル領域に表示される。スクロールUIに制約があるため、以下のルールを厳守する。

## コードブロックの使い分け

- **長文の日本語テキスト（原稿の採用案・サンプル文章・引用・書き換え案など）をコードブロック（```\`\`\```）で囲まない**
  - コードブロック内は折り返しされず 1 行表示になるため、狭いパネルで見切れて読めなくなる
  - 代わりに **blockquote（`>`）** または **通常の段落** で提示する
- **コードブロックを使うのは以下のケースのみ**:
  - シェルコマンド・実行例（`bash` / `zsh`）
  - ソースコード（`python`, `typescript`, `json`, `yaml` など実コード）
  - 出力結果例（ログ・エラーメッセージ）
  - ファイルパス単独提示（インラインの `` ` `` で十分な場合はそちらを優先）

## 採用案・書き換え案の提示フォーマット

原稿の書き換え案やレビュー項目の Before / After を提示する場合は、blockquote を使う：

```
**採用案A**:

> 　CDKを初めて使用するAWSアカウント...以降のCDKデプロイは〜
```

- blockquote 内は通常テキストとして折り返し表示されるため、狭いパネルでも最後まで読める
- 段落のインデント（全角スペース）・Markdown の太字・バッククォートもそのまま blockquote 内で機能する

## 箇条書き・表の取り扱い

- 表（Markdown テーブル）は VS Code 拡張でレンダリングされるため、そのままトップレベルに書いてよい（コードブロックや blockquote に入れない）
- 箇条書き（`-` / `*`）はデフォルトで折り返しされるため問題なし

## 横幅の目安

- VS Code サイドパネルのターミナル幅は **40〜60 文字程度** と想定して書く
- 1 行を長くしすぎない（コードブロック内の横長コマンドは除く）
