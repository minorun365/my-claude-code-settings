# ローカルブランチの整理

カレントディレクトリのGitリポジトリにあるローカルブランチを確認し、mainにマージ済みで不要なブランチを削除します。

## 前提条件

- カレントディレクトリがGitリポジトリであること
- mainブランチにいること（別ブランチにいる場合はユーザーに確認）

## 実行手順

### Step 1: ブランチの状態を一覧化

```bash
git branch -vv
git branch --merged main
```

すべてのローカルブランチをリストアップし、以下の情報を整理する：
- ブランチ名
- リモート追跡ブランチとの差異（ahead / behind）
- mainにマージ済みかどうか

### Step 2: 削除候補と残留候補を分類

以下の基準でブランチを分類する：

**削除してよい（マージ済み）:**
- `git branch --merged main` に含まれるブランチ（main自身は除く）

**残すべき（未マージ）:**
- `git branch --merged main` に含まれないブランチ
- 進行中の作業ブランチの可能性があるため安全のため残す

分類結果をテーブル形式でユーザーに提示する。

### Step 3: マージ済みブランチを削除

```bash
git branch -d <ブランチ名1> <ブランチ名2> ...
```

- まず `-d`（安全削除）を試みる
- 「リモートにプッシュされていないがmainにはマージ済み」の警告が出た場合は、mainにマージ済みであることを確認の上 `-D` で強制削除する
- main以外の現在チェックアウト中のブランチは削除できないため、必要に応じて `git switch main` してから実行する

### Step 4: 結果を報告

削除したブランチと残したブランチの一覧を表示する。

## 注意事項

- **未マージのブランチは削除しない**。ユーザーが明示的に指示した場合のみ `-D` で削除する
- mainブランチは絶対に削除しない
- 削除前にユーザーへの確認は不要（マージ済みのものは安全に削除できるため）
- リモートブランチの削除（`git push origin --delete`）はこのスキルのスコープ外。ローカルのみ対象
